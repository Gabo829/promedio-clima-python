Promedio Semanal de Temperatura
----------------------------------------
Este repositorio contiene dos implementaciones en Python para calcular el promedio semanal de temperatura utilizando Programación Tradicional y Programación Orientada a Objetos (POO).
Ambas soluciones forman parte de una práctica para comparar ambos paradigmas de programación y comprender sus diferencias.

Objetivos de la Práctica
----------------------------------------
- Implementar un programa funcional utilizando estructuras tradicionales: funciones, listas y flujo lineal.
- Implementar una solución equivalente utilizando Programación Orientada a Objetos (POO): clases, atributos y métodos.
- Comprender conceptos como encapsulamiento, modularidad y organización del código.
- Comparar la estructura y estilo entre ambos paradigmas.

Programación Tradicional
----------------------------------------
La versión tradicional utiliza funciones para:

- Ingresar las temperaturas diarias.
- Calcular el promedio semanal.
- Mantener una estructura secuencial y modular sin clases.

Esta versión es más simple, fácil de entender y directa.

Programación Orientada a Objetos (POO)
----------------------------------------
En la versión orientada a objetos:

- Se define una clase ClimaSemanal.
- Los datos (temperaturas) están encapsulados.
- Se usan métodos para ingreso de datos y cálculo del promedio.
- Facilita escalabilidad y reutilización.

Esta versión representa mejor situaciones reales donde se gestionan datos y comportamientos dentro de objetos.

Comparación entre ambos enfoques
----------------------------------------

| Característica          | Tradicional                      | POO                   |
| ----------------------- | -------------------------------- | --------------------- |
| Organización del código | Funciones sueltas                | Clases y métodos      |
| Mantenimiento           | Más difícil en programas grandes | Más fácil y escalable |
| Encapsulamiento         | No aplicable                     | Sí                    |
| Reutilización           | Limitada                         | Alta                  |
| Complejidad inicial     | Baja                             | Media                 |
